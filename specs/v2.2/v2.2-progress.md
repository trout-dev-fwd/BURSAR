## Decisions & Discoveries

- **[Pre-implementation]**: GitHub repo renamed from `trout-dev-fwd/trout-accounting` to
  `trout-dev-fwd/BURSAR` (public). GitHub URLs are case-insensitive; code will use lowercase
  `trout-dev-fwd/bursar` per URL convention. Repo does not need to be public for release
  binaries (private repos support Releases with authenticated API calls), but public
  simplifies the update check (no auth needed) and enables unauthenticated issue URLs.

- **[Pre-implementation]**: Version numbering follows semver with `0.x.y` prefix indicating
  pre-1.0 status. `Cargo.toml` is the source of truth; git tags must match exactly.
  Bump to `v1.0.0` when Bursar is feature-complete for public release.

- **[Pre-implementation]**: Updates are forced (no skip option). If the download or verification
  fails, Bursar falls through gracefully and launches the current version with a warning.
  Rationale: simplifies UX code; bad-release risk is mitigated by SHA256 verification and
  developer control over the release pipeline.

- **[Pre-implementation]**: Feedback uses browser-based pre-filled GitHub issue URLs via
  `xdg-open` / `cmd /c start`. No GitHub PAT required. User types description in a
  multi-line TUI modal, then the browser opens with the issue pre-populated.

- **[Pre-implementation]**: Bug reports include last 5 audit log entries. Feature requests
  do not include audit entries. Rationale: convenience to user > convenience to dev;
  5 entries keeps URL length well within browser limits.

- **[Pre-implementation]**: `b` and `f` hotkeys for feedback are scoped to the `?` help
  overlay only ‚ÄĒ not global hotkeys. No conflicts with existing per-tab key bindings.

- **[Pre-implementation]**: New dependencies: `semver` (version comparison), `sha2`
  (checksum verification). Both are lightweight, well-maintained crates.

- **[Pre-implementation]**: Platform-specific restart: `exec()` on Linux (in-place process
  replacement), spawn-then-exit on Windows. Cleanup of `.old` binary happens on next launch.

- **[Pre-implementation]**: GitHub Actions workflow triggered by `v*` tags. Builds Linux
  and Windows x86_64 binaries. Generates `checksums.txt` with SHA256 hashes. macOS and
  ARM targets deferred.

- **[Pre-implementation]**: Progress bar style uses `‚Ėį` (filled) and `‚ĖĪ` (empty), 10 chars
  wide (each = 10%). Format: `‚Ėį‚Ėį‚Ėį‚Ėį‚ĖĪ‚ĖĪ‚ĖĪ‚ĖĪ‚ĖĪ‚ĖĪ [40%]` with status line above:
  `Updating to v0.2.2. . .` If `Content-Length` header is missing, show indeterminate
  pattern: `‚Ėį‚ĖĪ‚Ėį‚ĖĪ‚Ėį‚ĖĪ‚Ėį‚ĖĪ‚Ėį‚ĖĪ` (no percentage).

- **[Spec review]**: GitHub API requires `User-Agent` header or returns 403. All requests
  must include `User-Agent: bursar/{version}`.

- **[Spec review]**: Binary replacement must check for symlinks before proceeding.
  `std::env::current_exe()` resolves symlinks, so renaming at the resolved path would
  break the symlink. Detect and fall through with warning.

- **[Spec review]**: Binary replacement must pre-check write permissions on the binary's
  parent directory. If installed to a root-owned directory like `/usr/local/bin/`, the
  rename will fail. Test before downloading to avoid wasting bandwidth.

- **[Spec review]**: Windows restart path must explicitly restore the terminal (leave
  alternate screen, disable raw mode) before spawning the new process and calling
  `process::exit(0)`. On Linux, `exec()` replaces the process so the new binary
  re-initializes the terminal naturally.

- **[Spec review]**: Feedback (`b`/`f` hotkeys) is only available from the `?` help overlay
  in the Running state. Not available on the startup screen ‚ÄĒ intentional, startup screen
  is minimal and doesn't have a help overlay.

- **[Spec review]**: URL encoding for GitHub issue bodies uses the `urlencoding` crate.
  No hand-rolled encoder.

- **[Spec review]**: GitHub Actions workflow runs `cargo fmt --check`, `cargo clippy -D
  warnings`, and `cargo test` before `cargo build --release` on both platforms. Prevents
  releasing a build that doesn't pass verification.

- **[Phase 1, Task 2]**: No `trout-accounting` references found in active code (.rs, .toml,
  docs). The rename was completed in V2.1. Task 2 was verification-only ‚ÄĒ zero code changes
  required.

- **[Phase 1, Task 3]**: Version number in `render_banner_area` was `Alignment::Right`;
  changed to `Alignment::Center`. Affects both the startup screen and splash screen since
  they share `render_banner_area`.

- **[Phase 1, Task 4]**: `UpdateProgress` enum and `SplashState` struct added to
  `startup_screen.rs`. `render_splash` signature changed to accept `SplashState` instead of
